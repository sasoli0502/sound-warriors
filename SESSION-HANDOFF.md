## コンセプト

### タイトル
「Sound Warriors」

### 世界設定
- 音を使って戦わなければならなくなった世界
- アニメ・漫画的な世界観
- 主人公カイト（Kite）がサウンドウォリアーとして各地の敵と戦う
- 曲ごとにビジュアルがガラリと変わる

### コアの差別化
1. **ドーパミン設計** — パーフェクト時の爆発的な視覚・聴覚フィードバック
2. **ストーリー統合** — プレイしながら物語が進行
3. **曲ごとに変わる世界観** — ビジュアルが完全に切り替わる
4. **柔軟なステージ形式** — バトル（敵HPを削る）形式

## 技術仕様

| 項目 | 詳細 |
|------|------|
| **エンジン** | Kaboom.js v3000（CDN: `unpkg.com/kaboom@3000.1.17`） |
| **音声** | Web Audio API（プロシージャル生成、外部ファイル不要） |
| **構成** | 単一HTMLファイル（約685行） |
| **解像度** | 1080x600（16:9、stretch + letterbox） |
| **レーン幅** | 120px × 4レーン |
| **操作** | PC: D/F/J/K キー ｜ スマホ: レーンタップ（全画面高タッチゾーン） |
| **言語** | 英語 / 日本語（起動時選択 + メニューで切替可） |
| **モバイル** | 横画面強制（縦画面時は回転促進オーバーレイ表示） |

## 開発で得た知見（次のゲーム制作に活かす）

| 知見 | 詳細 |
|------|------|
| Kaboom.js v3000の`parent`予約語 | コンポーネントプロパティに`parent`は使えない。`noteRef`→`noteId`に変更で解決 |
| `tween()` APIの差異 | v3000の`tween()`は独自シグネチャ。`wait()` + 直接代入で代替 |
| Web Audio APIで十分 | 効果音は外部ファイル不要。オシレーターで判定音・覚醒音・勝利音を生成 |
| Canvas + Playwright | CanvasゲームはスナップショットにDOM要素が出ない。テストはキーボード操作で |
| 単一ファイル戦略 | プロトタイプは1ファイルで十分。685行程度なら管理可能 |
| iOS横画面強制 | Screen Orientation APIはiOS Safariで効かない。JSでの縦横検出+オーバーレイが確実 |
| 16:9が標準 | モバイル横画面にフィットさせるには4:3ではなく16:9（1080x600）を使う |
